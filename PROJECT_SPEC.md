## 一、项目概述

### 1.1 项目名称

**24小时自动发卡平台**（代号：AutoCard）

### 1.2 项目定位

一款 **SaaS 化的虚拟商品自动发卡平台**，主要面向以下场景：

- 软件激活码、序列号、兑换码的售卖
- 会员账号（视频会员、网盘会员、学习平台会员）代充
- 游戏点卡、月卡、季卡、年卡的自动发货
- 各类兑换码、优惠券、礼品码的自动发送

### 1.3 核心价值

- **24 小时无人值守**：买家下单支付成功后，系统立即从卡密池自动分配卡密并发送至买家邮箱，**全程无需人工介入**
- **多规格销售**：同一商品支持多种规格（如月卡/季卡/年卡/永久版）独立定价、独立库存
- **卡密池管理**：商家可批量导入卡密，系统自动去重、自动分配、自动标记已售
- **完整的订单与卡密追溯能力**：支持买家自助查单、商家后台全程管理

### 1.4 商业模式

参考成熟平台 [yufaka.cn](https://www.yufaka.cn/)：
- 商家入驻 → 发布商品 → 导入卡密 → 买家自助下单 → 平台抽佣（可选）
- 也可作为单租户版本，平台与商家为同一主体

### 1.5 参考产品

| 参考对象 | URL | 借鉴点 |
|---------|-----|--------|
| 雨发卡 | https://www.yufaka.cn/ | 后台功能架构、自动发卡流程、卡密管理 |
| Xkun 卡片链接 | https://www.xkun.cn/links/419566DC | 商品详情页布局、暗色卡片视觉风格 |

## 二、术语与角色

### 2.1 术语表

| 术语 | 解释 |
|------|------|
| **卡密** | 商家导入的可被售出的字符串（如 `XXXX-XXXX-XXXX-XXXX`），是平台的核心资产 |
| **规格 (SKU)** | 同一商品下的不同选项（如月卡、季卡、年卡），每个规格独立定价/独立库存 |
| **订单** | 买家的一次购买行为记录，包含商品、规格、数量、金额、邮箱、状态 |
| **自动发卡** | 订单支付成功后，系统自动从对应规格的卡密池中取出一条未售卡密，绑定到订单，并发送至买家邮箱 |
| **卡密池** | 所有未售卡密的集合，按 `商品ID + 规格` 维度隔离 |
| **回收站** | 删除的商品/订单/卡密暂存处，30 天内可恢复 |

### 2.2 角色定义

| 角色 | 描述 | 入口 |
|------|------|------|
| **游客** | 未登录的访问者，可浏览商品、下单 | 前台 |
| **买家 (Customer)** | 完成下单并支付的用户（无需注册，通过邮箱识别身份） | 前台 |
| **商家管理员 (Admin)** | 商品发布、卡密管理、订单处理、店铺运营 | 后台 |
| **超级管理员 (Super Admin)** | 多店铺管理、权限分配（本期可选） | 后台 |

## 八、核心业务流程

### 8.1 自动发卡核心流程（最重要）

```
┌─────────┐  选择规格    ┌─────────┐  提交订单   ┌─────────┐
│  买家   │────────────→│ 结算页  │───────────→│ 创建订单 │

                                                      │ status=待支付
                                                      │ 锁定1条卡密（可选）
                                                      ▼
┌─────────┐  扫码支付    ┌─────────┐  支付回调   ┌─────────┐
│ 买家    │────────────→│  支付页  │←───────────│  第三方  │
└─────────┘             └────┬────┘            │  支付    │
                             │                  └──────────┘
                             │ 3秒轮询
                             ▼
                      订单状态变更为「已支付」
                             │
                             ▼ 自动执行 ↓
                  ┌────────────────────────┐
                  │ 1. 从卡密池取一条未售    │
                  │ 2. status=已售          │
                  │ 3. 绑定 order_id        │
                  │ 4. 库存 -1              │
                  │ 5. 发送邮件             │
                  │ 6. 订单状态=已发卡       │
                  └────────────────────────┘
                             │
                             ▼
                      买家可见卡密
```

### 8.2 卡密分配逻辑（伪代码）

```python
def assign_card(order):
    """事务：分配卡密并标记订单"""
    with db.transaction():
        # 1. 悲观锁查询可售卡密（FOR UPDATE）
        card = db.query(Card).filter(
            Card.product_id == order.product_id,
            Card.spec_id == order.spec_id,
            Card.status == AVAILABLE
        ).with_for_update().first()

        if not card:
            # 无库存
            order.status = REFUND_PENDING  # 待退款
            send_notify_to_admin('库存不足', order)
            raise OutOfStockError

        # 2. 标记卡密为已售
        card.status = SOLD
        card.order_id = order.id
        card.sold_at = now()

        # 3. 更新订单
        order.status = DELIVERED
        order.delivered_at = now()

        # 4. 解密卡密（DB 加密，前端展示用）
        plain_card = decrypt(card.content)

    # 5. 事务外异步发邮件
    send_card_email.delay(order.buyer_email, order, [plain_card])

    return plain_card
```

### 8.3 订单超时关闭流程

```
定时任务（每分钟执行）：
  ↓
查询 status=待支付 AND expired_at < NOW() 的订单
  ↓
逐个处理：
  - 订单 status=已关闭
  - 若有锁定卡密，释放回 AVAILABLE
  - 释放库存
  ↓
推送通知给买家（可选，提示订单已过期）
```

### 8.4 退款流程

```
买家发起退款（仅特定时间内）
  ↓
商家审核（后台点击"同意退款"）
  ↓
调用第三方支付 API 退款
  ↓
退款成功后：
  - 订单 status=已退款
  - 卡密 status=作废（不回收，避免重复使用）
  - 库存 +1（如有需要）
  ↓
通知买家退款成功
```

## 十二、安全性需求

### 12.1 数据安全

- 卡密内容在数据库中 **必须加密存储**（AES-256）
- 卡密传输到买家邮箱时使用 TLS
- 密码使用 bcrypt（cost=12）加密存储
- 敏感配置（支付密钥、邮件密码）使用环境变量或密钥管理服务

### 12.2 接口安全

- 全站 HTTPS
- 后台接口必须登录（JWT/Session），Token 有效期 2 小时，刷新机制
- 防 SQL 注入：使用 ORM / 参数化查询
- 防 XSS：用户输入转义、富文本使用白名单
- 防 CSRF：后台表单提交带 Token
- 接口限流：登录 5次/分钟/IP，发送邮件 1次/分钟/订单

### 12.3 业务安全

- 订单创建后立即锁定卡密（防超卖）
- 支付回调必须校验签名（防伪造）
- 订单查询需邮箱 + 订单号双重校验（防撞库）
- 后台登录失败 5 次锁定 10 分钟
- 操作日志：所有后台操作记录（谁、何时、改了什么）
